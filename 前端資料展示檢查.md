# knowledge-map 前端資料展示檢查

日期：2026-05-20

## 檢查目的

確認 `knowledge-map/data/units.json` 經過 indicator 品質校對後，仍能被 `knowledge-map/map.js` 穩定讀取與展示，避免資料正確但前端斷線、篩選失效或主線標示對不到單元。

## 檢查結果

| 項目 | 結果 | 備註 |
|---|---:|---|
| `map.js` 語法 | 通過 | `node --check knowledge-map\map.js` 無錯誤 |
| `units.json` 單元數 | 111 | 一到六年級上下學期完整 |
| 有 indicators 的單元 | 111 | 全單元皆有課綱代碼 |
| indicator 對應總筆數 | 202 | 第三批高頻校對後的目前狀態 |
| 不重複 indicator code | 139 | 移除部分明顯誤配後的目前狀態 |
| 必要陣列欄位缺漏 | 0 | `objectives`、`indicators`、`activities`、`prerequisites`、`successors`、`tools` |
| indicator 空白 code / text | 0 | 可支撐篩選選單 |
| 疑似截斷 indicator 文字 | 0 | 以短字串與開頭標點做初步檢查 |
| prerequisites 斷鏈 | 0 | 所有先備單元 ID 均存在 |
| successors 斷鏈 | 0 | 所有後續單元 ID 均存在 |
| `STRANDS` 單元 ID | 111 | `GxA/B-Uy` 清單完整解析 |
| `STRANDS` 無法對應單元 | 0 | 主線色彩可對上 `units.json` |
| 小數副線缺漏 | 0 | `G4B-U7`、`G5B-U8`、`G6B-U1` 已在小數線 |

## 目前可相信的前端功能

- 單元節點能依 `grade`、`semester`、`unit_number` 排列。
- 先備 / 後續關係不會因斷鏈造成空節點。
- 課綱代碼篩選選單可從 139 個 code 建立。
- 點擊 indicator code 後，可以找到對應單元群。
- 主線色彩標示可以從 `STRANDS` 對應回 111 個單元。
- 小數線已包含分數或比率單元中的小數副線。
