# knowledge-map 資料來源說明

## units.json 怎麼來的

### 第一層：骨架自動抽取

`../scripts/extract_units.py` 從教冊資料篇 PDF 抽出全年級（1～6年）單元名稱、學期、單元號，產生草稿 JSON。

來源 PDF（位於 `../數學學習地圖/`）：
- `114(1)數學1上～6上教冊(資料篇).pdf`
- `114(2)數學1下～6下教冊(資料篇).pdf`（三下使用教學篇補充）

### 第二層：三下手動補充

`../scripts/build_units.py` 將草稿與三下手動資料合併，輸出 `data/units.json`。

三下 9 個單元（U1～U9）的以下欄位是手動補充的：
- `activities`：從教冊活動名稱手動整理
- `tools`：連結到本資料夾的互動教學工具
- `notes`：備課提醒

### 第三層：全年級 objectives 與 indicators 補充

`data/units.json` 後續已補入全年級 111 個單元的 `objectives` 與 `indicators`：

- `objectives`：依教冊資料篇「教材分布」整理，已補滿 111 / 111 單元
- `indicators`：由教師手冊資料篇「課程綱要內容對照表」自動萃取，再進行人工校對，已補滿 111 / 111 單元
- 2026-05-20 第一批品質校對已修正明顯跨欄錯配與截斷文字，詳見 `可疑indicators校對清單.md`

### 第四層：先備/後續關係校正

全年級 111 個單元的 `prerequisites` / `successors` 來自教冊資料篇「**教材地位**」欄位，手動逐一校對。

校對過程紀錄在 git：
- `fix: 修正 3年下 6 個單元先備關係（來源：教冊資料篇教材地位）`
- `fix: 修正全年級先備關係（來源：教冊資料篇教材地位欄）`
- `fix: 手動驗證並修正 9 個未被程式解析的單元先備關係`

### 關於「合理推斷」的說明

`build_units.py` 初版的三下先備/後續是手動推斷寫入，後來全部依教冊資料篇重新校正，主要修正如下：

| 單元 | 初版推斷 | 校正後（教材地位） |
|---|---|---|
| 分數的加減 | 2下分數 ＋ 3上分數 | 只保留 3上分數 |
| 除法 | 乘法 ＋ 分分看 ＋ 3上除法 | 只保留 3上除法 |
| 尋找規律 | 1上排順序比多少 | 改為 2上加減關係與應用 |
| 公斤和公克 | 後續有「立方公分」 | 移除（教材地位欄空白） |
| 小數 | 先備有「分數的加減」 | 改為 3上分數 |
| 圓 | 先備有「角」 | 改為 1上方盒圓罐球 |
| 時間 | 先備有三個單元 | 只保留 2上幾時幾分 |
| 統計表 | 先備有「圖形與分類」 | 移除（教材地位欄無此列） |

**目前 `units.json` 的先備/後續是校正後的版本，可直接引用教冊資料篇為來源。**

## 主線（STRANDS）的來源

`map.js` 裡的 `STRANDS` 常數（14 條主線）是獨立維護的，**不從 units.json 讀取**，來源是教冊資料篇的數學學習主軸分類，搭配手動判斷歸線。
